Инструкция по подключению и использованию БД в проекте.

1. Требования перед началом работы: 
postgresql (как минимум серверный компонент) не ниже версии 9.6
pgadmin или любой другой совместимый клиент для облегчения взаимодействия с БД.

Если экземпляр postgres уже был настроен, имеется созданная отдельно БД с административной учетной записью, то следует перейти сразу к пункту 4 инструкции.

2. Настройка СУБД

Процесс инсталляции достаточно типовой, поэтому процесс установки и первоначальной настройки опускается. 
В процессе инсталляции компонентов postgresql создается учетная запись администратора и пароль, которые необходимо сохранить для продолжения работы с БД. Что необходимо сохранить перед работой:
 - имя учетной записи администратора, созданного при установке postgresql
 - пароль учетной записи администратора, созданного при установке postgresql
 - номер порта для соединения с БД, указываемый при установке postgresql

3. Создание и настройка БД

Не рекомеднуется использовать стандартную учетную запись postgres для операций, поэтому в пункте 3 рассмотрим создание новых базы данных и отдельной административной учетной записи.

Для управления базой данных необходимо запустить оболочку через GUI-клиент (например pgadmin) или интерфейс командной строки и осуществить вход. Пример для терминала:

psql -U postgres
где psql - это оболочка, -U - аргумент для указания пользователя, postgres - логин для подключения. 
После ввода будет запрошен пароль пользователя postgres, завершается вводом Enter. 
В случае возникновения проблем в первую очередь следует проверить, что служба сервера postgresql запущена, а также корректность ввода логина, пароля и адреса сервера с портом в случае удаленного подключения к СУБД. Также при удаленном подключении должны быть разрешены входящие подключения в конфигурационных файлах postgresql, настройки брандмауэра/антивируса должны разрешать подключение по соответствующему порту.

Для дальнейшего взаимодействия с БД можно использовать стандартную учетную запись postgres, но рекомендуется создать новую. Процесс создания учетной записи описан в отдельных руководствах, для запуска программы потребуется сохранить имя пользователя и пароль, а также выдать разрешение на полный контроль над базой данных, которая будет создана на следующем шаге.

Для создания новой базы данных в случае pgadmin необходимо установить курсор на имени сервера, открыть контекстно меню, пункты Create -> Database..., указать имя для новой базы данных и учётную запись-владельца базы данных, созданную на предыдущем шаге.
Через интерфейс командной строки база данных создается командой
CREATE DATABASE dbname;
ALTER DATABASE dbname OWNER TO new_owner;
где dbname - имя для новой базы данных, new_owner - учетная запись, созданная на предыдущем шаге.

4. После выполнения всех действий должны быть получены имя базы данных, логин и пароль учетной записи для администрирования базы данных и информация для подключения к БД (адрес и порт хоста с сервером postgresql).

Все данные необходимо указать в файле psql_connect.py в одинарных кавычках за исключением параметра LOGGING.

Описание параметров:

DBNAME = 'dbname' - параметр отвечает за имя базы данных, к которой производится подключение, краткое описание приведено в пункте 3.

DBUSER = 'admin' - имя пользователя-администратора базы данных, указанной в DBNAME.

DBPASSWORD = 'admin_password' - строка с паролем пользователя-администратора базы-данных.

DBHOST = 'localhost' - адрес сервера базы данных (машина с установленным сервером postgresql). Допускается указание IP-адреса.

DBPORT = '5432' - порт сервера базы данных, при установке postgresql по умолчанию используется порт 5432.

LOGGING = "DEBUG" - уровень логирования скрипта. Указывается между двойными кавычками, допустимые значения - "ERROR", "DEBUG". При указании уровня "DEBUG" в лог выполнения программы будет попадать дополнительная диагностическая информация.
